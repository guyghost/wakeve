## Apple Source Baseline

Last checked: 2026-05-28.

- Apple says an app record must be created before a build can be uploaded to App Store Connect.
- Apple says the required role to upload builds is Account Holder, Admin, App Manager, or Developer.
- Apple says apps can be uploaded using Xcode, Swift Playground, altool, or Transporter.
- Apple says App Store Connect accepts upload from supported Xcode versions and currently requires Xcode 14 or later for upload.
- Apple says build delivery progress, warnings, errors, and delivery logs are available for uploaded builds.
- Apple says a build must be processed in Apple's system before it appears in App Store Connect.
- Apple says email is sent when build processing is complete.
- Apple says each uploaded build is associated with the app and version record using the bundle ID and version number in the app bundle.
- Apple says uploaded builds can be distributed for testing or submitted for review after they are available in App Store Connect.
- Apple says the app's build string uniquely identifies the build throughout the system.
- Apple says uploaded builds can be viewed by version number, and build metadata can be inspected from the TestFlight tab.
- Apple says build metadata includes compressed file size, essential content size, additional in-app content size, and other build-related information.
- Apple says build upload status shows version, build number, upload status, and creation date.
- Apple says uploaded builds may go through app thinning, and final App Store DRM/recompression can change the final customer-facing size.
- Apple says iOS, iPadOS, macOS, tvOS, and visionOS builds plus each Mach-O executable must stay within maximum build file size limits.
- Apple says Xcode archives gather app binaries and dSYM files for distribution.
- Apple says TestFlight or App Store distribution can include symbol files when uploading the app to App Store Connect.
- Apple says dSYM files are needed to view symbolicated logs and crash reports.
- Apple says dSYM files generated by App Store Connect, when available, can be downloaded from the build metadata page, but dSYM downloads are no longer available for submissions from Xcode 14 or later.
- Apple says preparing an app for distribution requires a team that belongs to the Apple Developer Program for TestFlight or App Store distribution.
- Apple says apps should be tested thoroughly before distribution to beta testers or release users.

## Closure Rule

Set `APP_STORE_RELEASE_ARTIFACT_EVIDENCE_COMPLETE=true` only after:

- The signed IPA hash, archive, dSYM UUIDs, release commit, and App Store Connect uploaded build number all refer to the same candidate build.
- The build was uploaded with `submit_for_review: false` and `automatic_release: false`.
- Upload delivery warnings/errors are reviewed, and App Store Connect processing is complete before the build is selected.
- The app executable and embedded Mach-O executable sizes remain within Apple maximum build file size limits.
- The release owner can reproduce which binary was submitted and where its symbols are retained.
- The final audit has been run with `./scripts/app-store-submission-audit.sh --check-live-urls --run-submission-ready`.
