# Wakeve Voice and Tone

## Voice
Wakeve sounds human, calm, warm, and clear. On iOS, it should feel like a capable friend helping a group get organized, not a corporate workflow tool and not a playful toy.

## Rules
- Prefer clear action labels over cleverness.
- Use warmth in content moments, not in every system control.
- Keep errors direct and actionable.
- Use short sentences on compact surfaces.
- Avoid pressure, hype, and childish language.

## UI Layer Copy
The UI layer should stay conventional:
- `Ajouter`
- `Modifier`
- `Supprimer`
- `Partager`
- `Rechercher`
- `Fermer`
- `Retour`

These labels should not be replaced by branded phrases.

## Content Layer Copy
The content layer can carry Wakeve personality:
- `Créer un moment`
- `Inviter le groupe`
- `Préparer la soirée`
- `Le moment prend forme.`
- `Tout le monde trouve sa place.`
- `La table se prépare.`
- `Le groupe se met en route.`

Use these only when context makes the meaning obvious.

## Empty States
Good empty states:
- explain what is missing
- keep the tone calm
- offer one clear next action
- avoid feature tours

Example:
`Aucun moment à venir. Lance une date et invite le groupe quand tu es prêt.`

Primary action:
`Créer un moment`

## Errors
Errors should be plain:
- what happened
- what the user can do next
- whether work is saved or queued

Example:
`Impossible de synchroniser pour l'instant. Tes changements restent enregistrés sur cet appareil.`
